# SKED Reference Layer — Orientation

You've just landed in a SKED-side pod. This directory is your starting point for understanding how SKED actually operates — tooling, people, process, and vocabulary. Read these in order:

1. **[[SKED Tooling Map]]** — where things live (GitLab, Notion Builds, Intercom, Monday.com, Vimeo, Figma) and how they cross-reference each other. The single most useful page on your first day.
2. **[[SKED Stakeholder Map]]** — who's in each pod, who runs what, and which Adio-level execs are shared with the ChiroHD side.
3. **[[SKED Process Notes]]** — the structured PDLC has been retired; each pod sets its own process. Read this before your first pod meeting so you don't expect rituals that aren't there.
4. **[[SKED Glossary]]** — terms you'll hear in Slack and standups (Alina, Drip, Builds, ARIA, etc.).

## Why this layer is lighter than `chirohd/`

SKED reference content is intentionally seed-only. The ChiroHD reference layer was built over months of pod work; SKED gets a four-file starter kit. The first SKED pod is expected to extend it as patterns emerge — promote anything you keep re-explaining into a wiki page or a new reference file. Where you see "TBD per first SKED pod" or "verify with current SKED PM," that's an invitation to fill in the gap as you learn.
