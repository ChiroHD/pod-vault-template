# Demo Pod — Reference Implementation

This directory will hold a fully-populated example pod vault showing what a healthy, mature vault looks like after several weeks of active use. Useful for:

- New PMs who want to see the end state before they start
- Reference for what good wiki pages, decision records, and meeting notes look like
- Examples of the cross-link density a mature vault develops

**Status:** Placeholder. Demo content gets populated in Phase 3 of the template build.

When populated, this dir mirrors the top-level pod-vault structure:

```
examples/demo-pod/
├── shared/wiki/          # 5-8 example wiki pages spanning all categories
├── projects/
│   └── demo-project/     # One fully-developed project
│       ├── plan.md       # Filled-in plan with current state + phases + decisions
│       ├── project-brief.md
│       ├── tasks/tasks.md  # Active tasks with realistic statuses
│       ├── decisions/    # 5-10 example decisions
│       ├── meetings/     # 5-10 example meetings
│       └── raw/          # Redacted example transcripts
├── log.md                # Several weeks of timeline entries
└── INDEX.md              # Reflects the populated state
```

The content is synthetic but realistic. Treat as a learning artifact, not a real project.
